# PRD：阶段 4 桌面体验与稳定性打磨

Status: implemented

## Problem Statement

阶段 0-3 已经完成编辑、写作体验、技术文档预览和 Tauri 原生读写基础，但桌面应用还缺少日常使用所需的稳定性护栏：菜单快捷键、拖放打开、窗口状态恢复、外部文件变化提示、保存失败恢复和大文件降级策略。

## Solution

在 Tauri 侧补齐 Windows-first 原生菜单、菜单事件、文件状态查询、历史快照列表和窗口状态持久化；在 Svelte 侧监听菜单命令和桌面拖放事件，打开文件前保留恢复副本，保存前创建快照，保存失败时不清除编辑内容，并对外部修改、文件删除/移动、只读文件和大文件给出明确状态反馈。

## Acceptance

- Windows 原生菜单提供打开、保存、另存为、撤销、重做、源码模式、主题和专注模式快捷入口。
- 拖放 `.md` / `.markdown` / `.txt` 到桌面窗口可打开文件；有未保存修改时不会静默覆盖当前内容。
- 窗口位置和尺寸能跨启动恢复。
- 文件被外部修改、删除、移动或只读时有明确提示。
- 保存失败时保留 `localStorage` 恢复副本，不丢失编辑内容。
- 大文件进入只读源码模式，避免语义解析阻塞界面。
- `pnpm check`、`pnpm test`、`cargo check` 和 Tauri no-bundle 构建通过。
