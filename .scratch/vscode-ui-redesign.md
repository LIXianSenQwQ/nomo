# Nomo UI 重构 PRD

## 1. 背景与目标

为了改善编辑器的桌面体验，将目前的布局重新设计为类似 VS Code 的轻量桌面应用界面。增强长文本的可读性，精简不必要的多余列，使应用更加紧凑、高效。

## 2. 界面设计规范与布局结构

整改后的界面整体分为**左侧资源管理器**与**右侧完整编辑内容区**。

### 2.1 左侧资源管理器 (Explorer)

- **功能范围**：只显示当前打开的文件夹、项目文件树、Markdown 文件，以及最近打开的文件入口。
- **移除项**：完全移除在此处的任何大纲视图或多余的分类侧边栏，减少视觉噪声。
- **视觉风格**：标准 VS Code Explorer 的浅灰底色（`#f3f3f3`），带 1px 极细边框，图标对齐，行高紧凑。

### 2.2 右侧内容区 (Right Main Area)

右侧区域为一个完整的容器，包含顶部的菜单栏、标签页、工具栏以及编辑器主体。

- **正文视口居中**：
  - 编辑器主体（`.editor-content-wrapper`）设定 `max-width: 800px`，宽度 100%。
  - 正文需要在**整个右侧内容区**的总宽度下实现视觉居中，而不是剔除大纲后的剩余宽度。
- **轻量内嵌大纲**：
  - 文档大纲不再作为独立侧边栏呈现，而是以绝对定位的辅助悬浮组件（`.outline-embedded`）放置在右侧内容区的最右侧。
  - 视觉无强分割线边界，背景可使用半透明白色并伴随毛玻璃效果（`backdrop-filter: blur(4px)`），悬浮融入背景。
  - 点击大纲标题，正文仍能触发跳转与高亮响应。

### 2.3 配色与强调色

- 采用极简浅色主题：主要背景为 `#ffffff`，面板背景为 `#f3f3f3`，细边框色为 `#e1e1e1`。
- 主强调色：绿色/青绿色（`#4CAF50` 或 `#2E7D32`），用于高亮当前激活的标签页、选中的树项与状态按钮。

### 2.4 状态栏 (Status Bar)

- 位于窗体最底部，使用 `#f3f3f3` 背景与 `#e1e1e1` 顶边框，内部水平排列“已同步”、“版本号”、“字数统计”与“Markdown”类型标识等，高 22px。
