# PRD：标签页、资源管理器与文档大纲联动修复

Status: needs-triage
Labels: needs-triage

## Problem Statement

用户在资源管理器中浏览当前工作区时，顶级目录的缩进比子目录更靠右，破坏了文件树的层级认知。点击目录中的 Markdown 文件后，应用能创建新标签页，但标签页正文没有稳定刷新到被点击文件，手动切换标签页时也会出现内容和标签状态不同步。文档大纲需要跟随当前标签页内容，并提供一个位于标签页上方操作区的隐藏入口。

## Solution

资源管理器应以当前打开文件夹作为根节点，根节点位于最左侧，子目录和文件逐级右移。打开 Markdown 文件、切换标签页、保存后更新标签页时，应用应显式同步当前标签页的 Markdown、脏状态、只读状态、文件路径和派生信息。文档大纲从当前标签页 Markdown 派生，并在标签页工具区提供显示/隐藏按钮，同时保留查看菜单中的文本入口。

## User Stories

1. As a Markdown 写作者, I want 资源管理器的顶级目录显示在子目录左侧, so that 我能一眼判断目录层级。
2. As a Markdown 写作者, I want 点击资源管理器里的 `.md` 文件后新标签页立即显示该文件内容, so that 我不需要重新切换或刷新。
3. As a Markdown 写作者, I want 手动切换标签页时正文、文件名、保存状态和统计信息同步变化, so that 我能放心同时打开多个文档。
4. As a 长文档作者, I want 文档大纲跟随当前标签页, so that 我在多个文档之间切换时仍能看到对应标题结构。
5. As a 长文档作者, I want 在标签页上方快速隐藏或显示文档大纲, so that 窄窗口或专注写作时可以释放编辑空间。
6. As a 键盘用户, I want 大纲显示按钮具备清晰的可访问名称和焦点状态, so that 我可以不用鼠标完成切换。

## Implementation Decisions

- 修改应用层的标签页状态同步逻辑，避免只原地修改标签页对象而不触发界面更新。
- 为编辑器设置 Markdown 增加“切换标签页”语义，避免打开或切换干净文件时被误标记为未保存。
- 资源管理器保留根节点与递归子树的分层模型，调整缩进基准而不改变 Tauri 文件树接口。
- 文档大纲继续由 Markdown 文本派生，不写入主存储，也不与资源管理器合并。
- 大纲显隐作为界面状态处理，不改变 Markdown 内容和标签页持久数据。

## Testing Decisions

- 优先验证 `pnpm check`，覆盖 Svelte/TypeScript 类型约束和组件绑定问题。
- 继续运行现有单元测试，确认 EditorCore、MarkdownBridge、Outline 与渲染服务没有回归。
- 对界面层进行手动验证：打开文件夹、点击 Markdown 文件、切换标签页、观察大纲内容和隐藏按钮状态。

## Out of Scope

- 不新增多窗口标签持久化。
- 不调整 Tauri 文件树扫描规则。
- 不实现大纲滚动跟随光标位置。
- 不引入新的 UI 组件库或主题重设计。

## Further Notes

本修复遵循现有领域边界：资源管理器属于左侧文件导航区域，Outline 属于内容区内的跟随导航。
